
### **Слайд 10: Арифметические операции**
**Базовые операции:**
```csharp
1.  int a = 10 + 5;   // Сложение: 15
2.  int b = 10 - 5;   // Вычитание: 5
3.  int c = 10 * 5;   // Умножение: 50
4.  int d = 10 / 3;   // Деление целочисленное: 3
```

**Сокращённые операции:**

| Операция | Пример | Эквивалент | Описание |
|----------|--------|------------|----------|
| Инкремент | `x++` | `x = x + 1` | Постфиксная форма |
| | `++x` | `x = x + 1` | Префиксная форма |
| Декремент | `x--` | `x = x - 1` | Постфиксная форма |
| | `--x` | `x = x - 1` | Префиксная форма |
| Сложение | `x += 5` | `x = x + 5` | Увеличивает на 5 |
| Вычитание | `x -= 3` | `x = x - 3` | Уменьшает на 3 |

**Важно:**  
Для деления с дробной частью нужен тип `double`:
```csharp
1.  double result = 10.0 / 3; // Результат: ~3.333...
```

### **Слайд 11: Разница между ++x и x++**
**Постфиксная форма `x++`:**  
Сначала используется значение, потом увеличивается.

**Префиксная форма `++x`:**  
Сначала увеличивается, потом используется значение.

```csharp
1.  int a = 5;
2.  int b = a++; // b = 5, потом a = 6
3.  
4.  int c = 5;
5.  int d = ++c; // c = 6, потом d = 6
6.  
7.  Console.WriteLine($"a={a}, b={b}, c={c}, d={d}");
8.  // Вывод: a=6, b=5, c=6, d=6
```

### **Слайд 14: Типовые ошибки в условных операторах**
**Ошибка 1:** Точка с запятой после условия
```csharp
1.  if (number > 0); // ОШИБКА! Точка с запятой завершает оператор
2.  {
3.      Console.WriteLine("Положительное");
4.  }
5.  // Выполнится ВСЕГДА, независимо от условия
```

**Правильно:**
```csharp
1.  if (number > 0)  // Без точки с запятой!
2.  {
3.      Console.WriteLine("Положительное");
4.  }
```

**Ошибка 2:** Присваивание вместо сравнения
```csharp
1.  if (x = 5) // ОШИБКА! Присваивание, а не сравнение
2.  {
3.      // ...
4.  }
```

**Правильно:**
```csharp
1.  if (x == 5) // Два знака равно!

3.      // ...
4.  }
```

**Ошибка 3:** Сложные условия без скобок
```csharp
1.  if (a > b && b > c || c == 0) // Неясный приоритет
```

```csharp
1.  if ((a > b && b > c) || c == 0) // Явная группировка
```


**Правильно:**

### **Слайд 15: Метод `TryParse` для безопасного преобразования**
**Проблема:** `int.Parse()` вызовет ошибку, если строка не содержит число.

**Решение:** `int.TryParse()` – безопасный метод преобразования.

```csharp
1.  string input = "123";
2.  int result;
3.  bool success = int.TryParse(input, out result);
4.  // success = true, result = 123
5.  
6.  input = "abc";
7.  success = int.TryParse(input, out result);
8.  // success = false, result = 0
```

**Синтаксис:**  
`bool имя_переменной = int.TryParse(строка, out int переменная);`

* Возвращает `true`, если преобразование удалось  
* Результат записывается в переменную через `out`

### **Слайд 18: Операторы сравнения и логические операторы**

### **Слайд 19: Оператор выбора `switch` (классический)**
```csharp
1.  int day = 3;
2.  string dayName;
3.  
4.  switch (day)
5.  {
6.      case 1:
7.          dayName = "Понедельник";
8.          break;
9.      case 2:
10.         dayName = "Вторник";
11.         break;
12.     case 3:
13.         dayName = "Среда";
14.         break;
15.     default:
16.         dayName = "Неизвестный день";
17.         break;
18. }
```

**Важные правила `switch`:**  
1. Каждый блок `case` должен заканчиваться оператором перехода: **`break`**  
2. Блок **`default`** выполняется, если ни один `case` не подошёл  
3. Можно использовать с `int`, `char`, `string` и другими типами

### **Слайд 25: Типовые ошибки в циклах**
**Ошибка 1:** Бесконечный цикл `while`
```csharp
1.  int x = 0;
2.  while (x < 5) // x никогда не меняется
3.  {
4.      Console.WriteLine(x);
5.      // Забыли x++ - цикл выполняется бесконечно!
6.  }
```

**Правильно:**
```csharp
1.  while (x < 5)
2.  {
3.      Console.WriteLine(x);
4.      x++; // Увеличиваем счётчик
5.  }
```

**Ошибка 2:** Изменение коллекции в `foreach`
```csharp
1.  List<int> numbers = new List<int> {1, 2, 3};
2.  foreach (int num in numbers)
3.  {
4.      if (num == 2)
5.          numbers.Remove(num); // ОШИБКА! Нельзя изменять коллекцию в foreach
6.  }
```

**Решение:**
```csharp

2.  {
```

**Ошибка 3:** Использование `=` вместо `<=` в условии

   **Вывод:** `"Условие 3 выполнено"`

**Тема лекции 2: C#: Методы, массивы, основы ООП. Значимые и ссылочные типы. String**

5.  **Введение в ООП (минимально необходимое для алгоритмов):**
    *   Класс как новый ссылочный тип данных. Объект — экземпляр класса.

    *   Модификаторы доступа (`public`, `private`).
6.  **Практический пример:** Создание простого класса (например, `Student`), демонстрация работы со списком студентов и формирование отчета о них с использованием `StringBuilder`.
